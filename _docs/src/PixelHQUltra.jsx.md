## Overview

This file is a React .jsx module that composes an interactive multi-agent demo. It defines a module-level INITIAL singletons (EventBus, A2AProtocol, PersonalityEngine, TerminalBridge wrapper via imports) and a central useReducer-based application state (INIT + reducer). The UI surface is built from small functional React components (PixelCharacter, SpeechBubble, AgentSprite, MessageParticle, MeetingOverlay, OfficeWorld, HUD, TerminalFeed) and a top-level exported component PixelHQUltra which mounts the app, connects the terminal bridge, registers EventBus listeners, and sets periodic timers to drive animations and state expiration.

The file tightly couples UI rendering with game state and external events. Office map tiles are generated by generateOfficeMap() into OFFICE_MAP and painted as absolutely positioned divs; agents are kept in a keyed object (agents keyed by id) inside the reducer state and rendered by mapping over Object.values(agents). The file implements fog-of-war (state.revealed is a Set of "x,y" strings), particle animations (state.particles progressed by PARTICLE_TICK via setInterval), meetings/debates (meeting/debate sub-objects in state, plus automatic scripted sequences when GAME_MEETING_START events arrive), XP and evolution logic (XP_GAIN reducer case referencing XP_TABLE and EVOLUTION_MILESTONES), speech bubbles (ADD_BUBBLE + EXPIRE_BUBBLES), and a canvas-based Knowledge Graph overlay rendered by KGOverlay when toggled. Event wiring: TerminalBridge and EventBus emit events (TERMINAL_EVENTS.* and a2a messages) that dispatch reducer actions driving UI updates and simulated behaviors.

## Architecture Notes

- State management: The file uses a single reducer (function reducer(state, action)) and an INIT constant to hold application state. All UI updates originate from dispatching actions to this reducer (e.g., AGENT_MOVE, ADD_BUBBLE, PARTICLE_ADD, MEETING_START, XP_GAIN). This centralizes state mutations and makes effects predictable via reducer cases.
- Event-driven integration: The module-level singletons (bus, a2a, personas, kgOverlay, OFFICE_MAP) and TerminalBridge connect external terminal / A2A events to the reducer. The EventBus implements an observer/emit pattern where PixelHQUltra registers bus.on() handlers and translates them to dispatch calls; this is an Observer pattern for decoupled event handling.
- Rendering & performance patterns: The office world is rendered as many absolutely positioned divs (one per tile) and agent components layered on top. Fog-of-war uses a Set of 'x,y' strings and tile rendering toggles CSS filter for visibility—this is simple but could be costly for large WORLD_W/WORLD_H values. The knowledge-graph overlay uses a canvas and requestAnimationFrame rendering loop only when visible, minimizing CPU when hidden.
- Lifecycle & cleanup: PixelHQUltra's top-level useEffect connects the TerminalBridge, registers bus listeners, schedules multiple setInterval and setTimeout timers, and returns a cleanup that unsubscribes all listeners, clears intervals/timeouts, and disconnects the bridge. timeoutsRef is used to track timeouts for cleanup.
- UI composition: The file is organized as small pure functional components (stateless/presentation where possible) and the root PixelHQUltra that wires side effects. Many components accept the minimal props required: e.g., OfficeWorld({ state, dispatch }) and AgentSprite({ agent, isSelected, onClick }).

## Usage Examples

### Manual meeting call via UI button

Clicking the "📅 Call Meeting" button triggers bus.emit(TERMINAL_EVENTS.GAME_MEETING_START, meetingObject). The registered bus handler in PixelHQUltra's mount effect receives that meeting object and dispatches MEETING_START (populates state.meeting). It then iterates attendees and dispatches AGENT_MOVE and AGENT_STATE to position agents in the meeting room and set them to 'meeting'. It schedules MEETING_STATEMENT actions (and ADD_BUBBLE) progressively via setTimeouts to simulate a transcript, grants XP (XP_GAIN) and STAT_INC for participants, and finally dispatches MEETING_END after the scripted delay. The result: agents move, speech bubbles appear, the MeetingOverlay component shows transcript and attendees, and XP/evolution state may update.

### Terminal-driven work event

When a TerminalBridge or other external system emits TERMINAL_EVENTS.GAME_AGENT_WORK, the bus handler dispatches ADD_BUBBLE (work style), AGENT_STATE (working), STAT_INC (commandsRun), and XP_GAIN using XP_TABLE.tool_use. It also pushes an entry into the termFeed via TERM_FEED containing raw terminal text and a translation generated by personas.speak(...). TerminalFeed renders these entries, showing raw and translated text. The UI thus correlates raw terminal lines to in-game narrative and updates agent stats and visual indicators.

### A2A message particle and UI feedback

When an A2A message event arrives (bus.on(TERMINAL_EVENTS.GAME_A2A_MESSAGE) or bus.on('a2a:message')), the code dispatches PARTICLE_ADD to create a flying particle from sender to receiver and ADD_BUBBLE for sender with style 'a2a'. A timeout is scheduled so that after the particle arrival an ADD_BUBBLE is added on the receiver with 'Got it!'. Particles are animated by PARTICLE_TICK events (setInterval) that increase particle.progress until they expire, and MessageParticle computes a bezier arc for position based on agent positions.

## Maintenance Notes

- Performance: Rendering every tile as an absolutely positioned div (OFFICE_MAP.map(...)) can be expensive for large WORLD_W/WORLD_H. Consider virtualizing tiles, drawing static tile layers on a canvas, or only rendering tiles within the viewport with memoization to reduce DOM nodes.
- Timers and cleanup: The mount effect creates many setTimeouts and setIntervals and collects them in timeoutsRef and variables. Ensure any new timers are added to timeoutsRef or cleared in cleanup to avoid leaks. Keep the cleanup block up-to-date if you add other global resources.
- Concurrency & state consistency: Many bus handlers dispatch multiple actions in sequence; because useReducer updates are synchronous within React event handlers but asynchronous across timeouts, be careful when depending on current state values (e.g., agent lookups) inside delayed callbacks. Prefer reading stable values at scheduling time or using refs if necessary.
- Testing: Unit-test reducer behavior (each action type: AGENT_MOVE, ADD_BUBBLE, PARTICLE_TICK, XP_GAIN, etc.) and component rendering in isolation. Mock EventBus and TerminalBridge to exercise the event handlers in PixelHQUltra's mount effect. Add tests for KG overlay interactions (hitTest, canvas click -> selectedNode).
- Extensibility: Game rules (XP_TABLE, EVOLUTION_MILESTONES) are imported from officeData.js — to change progression behavior modify that module rather than the UI code. If you add new TERMINAL_EVENTS, add corresponding bus handlers in the mount effect and matching reducer cases if state mutation is needed.
